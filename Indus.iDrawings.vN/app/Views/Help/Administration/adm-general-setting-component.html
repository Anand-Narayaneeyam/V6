This option enables the Administrators to define a password policy for enhancing the security by imposing some conditions on the creation of passwords by the users. The following conditions are provided in the Password policy setup.

Minimum length of the password - The minimum length of the password can be specified using this option. This can be set from 8 to 30

Case sensitive - Through this option, it can be specified whether the password is case sensitive or not.

Requires at least one upper case character - If this option is checked, then the password entered should have at least one upper case character (capital letter)

Requires at least one lower case character - If this option is checked, then the password entered should have at least one lower case character (small letter).

Requires at least one non-alphanumeric character - If this option is checked, then the password entered should have at least one special character like @, #, $, *, etc.

Requires at least one numeric character - If this option is checked, then the password entered should have at least one numeric character (0-9)

Password expiry - Two options are provided for password expiry - Password Never Expires and Expire after (No. of) days.
If the set option is Password Never Expires then the password will never expire, otherwise the password will expire after the number of days entered in the text box. If the password has expired, the system will display ‘Your password has expired’ at the user’s next login and prompt the user to change the password.

Minimum time between password changes - This is the minimum time interval needed between two password changes. Once a user has changed his password, the user will be allowed to change his password again only after the specified time interval.

Number of previous passwords that cannot be reused – By specifying a number ‘n’ here, it can be ensured that when a user changes his password, it will be different from his past ‘n’ passwords.

Maximum number of consecutive failed attempts before the account gets locked - This option is used to lock the user’s account if the user has entered wrong passwords for the specified number of consecutive attempts within a set Time Interval.

Time Interval for counting failed attempts to lock the account - This is the Time Interval within which consecutive failed attempts can lock an account.

Account locked out period – This option is used to specify a time period for keeping a user account disabled once it gets locked due to repeated failed attempts to login as defined by the two options defined above. The locked user account will be enabled automatically after the lock period specified here has elapsed.

Account inactive – This option is used to set a policy for making the user account inactive if not logged in for a set period.
The Administrator can specify whether the user account has to be locked if the account is inactive for the specified number of days or Account Never locked if the account is inactive.
The locked user account will be enabled automatically after the Account inactive period specified in the policy has elapsed.

After a password policy change, when a user logs in, if his password does not comply with the new password policy, he will be prompted to change his password.

User Category:

This option is to add users of different category to a User Group. The default User Categories provided are iDrawings Users/Employees/Technicians/Contractors and these Categories will be displayed as tabs. User Categories will be based on the modules selected by the customer. Employees’ tab will be available only if Employee module is enabled. Technicians or Contractors tab will be available only if Work Order module is enabled for the Customer.

Select the required User Group to which the Users are to be added and click the User Category icon. All User Categories will be displayed as tabs. By Default iDrawings Users tab will be selected.

Add: Administrator can add users of User Category ‘iDrawings Users’ to the selected User Group by clicking the Add icon in the iDrawings Users tab. On clicking the Add button, iDrawings Users selection page will be displayed with the list of all active iDrawings Users (other than WO User) along with their Email and User Role. Select/Deselect the required users by checking/unchecking the checkboxes shown against each user. On clicking the Save Changes button in the iDrawings Users selection page the selected users will be added to the selected User Group.

Delete: Users added to the selected User Group can be removed through this option.

Likewise, users of different categories like Employees, Technicians and Contractors can be added to each group.

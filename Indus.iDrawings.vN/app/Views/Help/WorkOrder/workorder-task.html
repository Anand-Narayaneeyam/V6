Review of Service Requests and PM Work Orders are grouped under the Current Tasks menu.

Service Requests tab lists all the Service Requests that are pendingfor review by the logged in user, including those which have been converted to Work Orders, whereasPM Work Orders tab list all the PM Work Orders that are pending for review by the logged in user. For an action point user, all the requests/work orders which are to be reviewed by him will be displayed whereas for an administrator, all the requests/work orders will be displayed in the Service Requests / PM Work Orders tab. On clicking Current Tasks menu, Service Requests tab will be selected by default.

Service Requests

The Action Point user can review requests involving him/her. On clicking Current Tasks: Service Requests tab, all the Service Requests that are pending for review by the logged in user, including those which have been converted to Work Orderswill be listed. If the logged in user is a Customer Administrator or Module Administrator of Work Order module, requests/Work Orders to be reviewed by all other users also will be displayed. The Request will be displayed with all static fields. Action can be taken on these requests or work orders by selecting the desired request/work order and clicking the Review icon.

Option to create a new service Request is provided in the Service Requests list.

Once a Service Request is created, it will be available for review under Current Tasks: Service Requests tab for the first Action Point user, for all Customer Administrators and for all Module Administrators of Work Order module.

On reviewing, the user can select one of the actions defined for the Action list and if the action point is of Specific User type, the Next Action Point User(s) can also be selected. Multiple User selection is allowed for Next Action Point User. Once the action is confirmed by clicking the Save Changes button, the request will move to the next Action Point as defined in the workflow.

A Work Order is generated when a request is reviewed and submitted by selecting the action 'Generate Work Order'. Once a Work Order is generated, it will be available for further review in Current Tasks: Service Tasks for the next Action Point user and for all Administrators. (Only if further Actions Points and Outcomes are defined in the corresponding Workflow)

Review: User can select the required Request or Work Order to be reviewed from the list and click on Review icon.

Review Request page is displayed as a tab with Review Comments, Previous Comments, Request details, additional data fields based on the settings made under Settings: Work Order: General Settings: Set Workflow: Workflow Settings: Editable Fields, and Outcomes of current Action Point. If the Next Action Point is associated with more than one user, then there is an option to select user(s) for the Next Action Point. For this, the 'Specify User at Previous Action Point' should have been set through Administration: Action Points: Add/Edit.

If the Specify User option is not set, there will not be any option to select a User in the Review page. In this case, any one of the users associated with the selected Action Point can Review the request/Work Order and take necessary action.

Work Orders can be set to On Hold by entering the Hold details. When selecting the Work On Hold check box, a combo box under the Hold details will be enabled with reasons listed. The Reasons combo lists reasons such as Waiting for Building Engineer, Waiting for gathering information from Requester, Waiting for Government approval, Waiting for Materials, Waiting for Other Reasons and Waiting for Outside contractor. New Reasons can be entered by clicking the Add icon near the Hold Reason field. On selecting the reason for On Hold, a date time control will be displayed for selecting the start time of the selected reason. On changing the Reason, the end time of the previous reason will be taken as the start time of the current reason and On Hold Time will be calculated based on that.

The particular Work Order can be completed by selecting the outcome 'Complete Work Order' and clicking Save Changes button. On selecting the outcome Complete Work Order, Date of Completion field will be enabled and populated with the Current Date.

Note: Only Work Orders which are completed can be closed.

Option to enter the Time Spent (Hours) will be displayed in the Review Request page only if the "Enable Time Spent Editing" option is enabled by Indus.

Permissions - Manage Equipment, Manage Documents, Manage Invoices, Manage Cost and Manage Chargeback are provided in the Review Page. These options will be available only if the corresponding Permissions are provided for the current Action point of the selected Request/Work Order.

Manage Equipment: This link will be enabled only if the permission 'Manage Equipment' under Settings: Work Order: Workflow Settings: Set Workflow: Permissions for the particular Action Point is set and mapped against Service Requests Workflow Entity.

On clicking the Manage Equipment button, Equipment window will be displayed with a drop down to select the Equipment Category and with Add and Delete icons. Delete icon will be enabled only if any Equipment are linked to the selected Service Request or Work Order.

Add: Equipment can be added by clicking the Add icon after selecting the required Equipment Category. On clicking the Add icon, all the Equipment of the selected Category will be listed with check boxes. Users can select the Equipment from the list and add it to the Request.

Assigned Equipment will be displayed with Site/Building/Floor. Unassigned Equipment will be displayed without Site/Building/Floor. Clicking on the Site/Building/Floor, hyperlink will open the corresponding drawing, with the Equipment highlighted.

If Barcode option is enabled in any of the Object Module (Assets, Furniture etc), option to select image with valid barcode is provided. Attached Barcode will be decoded by the application. If any Equipment is linked with decoded barcode, it will be added to the selected Service Request.

Delete: Delete Equipment by selecting it and then clicking the Delete Delete icon.

Manage Documents: This link will be enabled only if the permission 'Manage Documents' under Settings: Work Order: General Settings: Set Workflow: Permissions for the particular Action Point is set and mapped against Service Requests Workflow Entity.

On clicking this link, Documents page will be displayed with an Add icon. Any document already attached to the Request will be displayed here.

Add: Documents can be added by clicking Add icon

Delete: Delete a document by selecting it and clicking on the Delete icon.

Download: Document can be downloaded and saved in the user’s computer by clicking the Download icon.

Manage Cost: This link will be enabled only if the permission 'Manage Cost' under Settings: Work Order: General Settings: Set Workflow: Permissions for the particular Action Point is set and mapped against Service Requests Workflow Entity. On clicking this link, Manage Cost page will be displayed with options to add Technicians, Parts, Tools & Other Costs. Technicians/Parts/Tools/Other Costs already added (if any) will be displayed with the cost details.

Technicians

Add: A new Technician can be added by clicking Add icon. Enter hours spent in the text box against Technician. Total Cost will be displayed at the bottom.

Delete: Delete a Technician by selecting it and then clicking Delete icon

Parts

Add: A new Part can be added by clicking Add icon. Option to select the equipment Category and Class is provided in New Part window so as to select the Parts associated with a particular Equipment Category or Class. Select the Part(s) and add them to the Parts grid in Manage Cost window, Enter Quantity in the text box against Part. Total Cost will be displayed at the bottom.

Delete: Delete a Part by selecting it and then clicking Delete icon.

Tools

Add: A new Tool can be added by clicking Add icon. Enter hours used in the text box against Tool. Total Cost will be displayed at the bottom.

Delete: Delete a Tool by selecting it and then clicking Delete icon.

Other Costs

Add: Other Cost can be added by clicking Add icon. On clicking Add icon, New Cost window will be displayed with Cost Item field. Enter the Cost Item and click Save button. The Cost Item will be added and displayed in Other Cost grid. Cost can be entered against the Cost Item. Total Cost will be displayed at the bottom.

Delete: Delete an Other Cost by selecting it and then clicking Delete icon.

Enter the required cost entries. Click the Save Changes button to save and close the cost details page. On deleting any Cost item, the cost already entered against the item will be deducted from the total cost.

Keyword Search

This option helps users to search for Service Requests or Work Orders using keywords. The keyword could belong to any data field of the Service Request. The user need not know or specify which field to look in. The Keyword could be any part of the Service Request viz. Work Type, Request Number, Requested Date, Previous Review Comments, Next Action Point etc. or the additional data fields.

On entering the keyword and clicking on the Search icon, the data grid will be filtered with Service Requests records having data matching the specified keyword.

Advanced Search

This option helps users to search for Service Requests or Work Order details with specific data in known datafields. The data being searched for can be selected from the list boxes or entered into text boxes provided against the data fields. The user can select one or more data fields viz. Work Type, Request Number, Requested Date, Previous Review Comments, Next Action Point etc. or the additional data fields. After entering the search criteria and click Search button, the data grid will be filtered with Service Requests or Work Orders which satisfy the selection criteria.

Fields to be displayed in the Search window can be set using the Configure icon, provided in the Search window.

Search criteria entered in the Search window can be cleared by clicking the Clear button provided in the Search window. After clearing the Search criteria and click Search button to clear the Search Results in the Service Requests grid.

More icon

Complete: This option enables the user to complete multiple requests in a single operation. For performing multiple complete operation of Service Requests, Work Orders following conditions are to be satisfied.

  1. The Current Action Point should have the outcome 'Complete Request'/'Complete Work Order' defined.
  2. If the selected Requests have the Next Action Point in 'Complete Request'/'Complete Work Order' outcome different, the action point should be of non-specific type (Specify User at Previous action point unchecked)
  3. On Hold Work Orders are not supported in multiple Complete.
  4. Only Work orders with same Status can be selected.

Select the Requests/Work Orders to be completed and click the Complete menu, Complete Service Requests/Complete Work Orders window will be displayed with Time Spent to complete. If Next Action point of the Complete Outcome is specific the users list box will be shown. Click the Save button after selecting User(s) and entering the time spent value, the selected requests/Work Orders will be completed and will be displayed in the grid with Completed Status.

Close: Close option enables the user to close multiple requests/Work Orders in a single operation, which are in Accepted or Completed status. For performing multiple Close operation of Service Requests/Work Orders following conditions are to be satisfied.

  1. If the requests are in Accepted status, both Complete Request/Complete Work Order and Close Service Request/Close Work Order outcome should be defined in the Current Action point, also Current and Next Action Point of the Complete Request/Complete Work Order Outcome should be the same.
  2. If the requests are in Completed status, Close outcome should be defined in the current Action point.
  3. On Hold Work Orders are not supported in multiple Close.
  4. Only Requests/Work orders with same 'Status' can be selected.

Select the requests/work orders to be closed and click the Close menu, Close Service Requests/Close Work Orders window will be displayed with Date of Completion and Remarks. Click the Submit button by entering the Date of Completion and Remarks, the selected requests will be closed. These requests will be available in reports of Closed Tasks.

Override: Override option enables the Administrator to override the workflow of the requests which are in a workflow. Sometimes the workflow might get stuck at some Action Point and the Administrator can move the request to another Action Point defined in the same workflow using this Override option.

By selecting the required request to override and on selecting the Override menu, an Override Request page will be displayed. Current Action point will be displayed as read only, Next Action Point and User list box will be displayed. Select a new Action point from the list. If the selected Action Point is associated with Specific User, User selection can be done. Click on Add button, the Service Request's/Work Order's Current action point will get changed to the newly selected Action point and an auto generated mail will be sent to the person responsible for the new Action point.

Reminder: Reminder option enables the Administrator to remind an Action Point user about a particular request. On selecting the required request to send reminder and on selecting the Reminder menu, Reminder message details will be displayed with default subject and message body. The Administrator can change the message details if required and click Send. The reminder email will be sent to the Action Point user.

PM Work Orders

The PM Work Order tab provides option for the users to manage the PM Work Orders.

PM Work Orders are generated based on the PM Schedules created for Equipments in Settings: Work Order: PM Settings menu. Once a PM Work Order is generated, it will be available for further review under Current Tasks: PM Work Orders for the next Action Point user and for all Administrators.(if further Action Points and Outcomes defined in the corresponding Workflow)

Only those Work Orders concerning the logged in user will be shown here. These are the PM Work Orders on which he/she has to take action. If the logged in user is an administrator or Module Administrator of Work Order module, all PM Work Orders to be reviewed by all other users also will be displayed. The PM Work Order will be displayed with all static fields. Action can be taken on these PM Work Orders by selecting the desired PM Work Order and clicking the Review icon.

On reviewing, the user can select one of the actions defined for the Action list and if the action point is of Specific User type, the Next Action Point User can also be selected. If the Action Point is associated with User Groups, only iDrawings Users belong to the group will be listed in the Select User combo. Once the action is confirmed by clicking the Save Changes button, the PM Work Order will move to the next Action Point as defined in the workflow.

Review: User can select the required PM Work Orderto be reviewed from the list and click on Review icon.

Review PM Work Order page is displayed with Review Comments, Previous Comments, Cost Details, Work Order details, additional data fields based on the settings made under General Settings: Set Workflow: Workflow Settings: Editable fields and Outcomes of current Action Point. If the Next Action Point is associated with more than one user then there is an option to select user(s) for the Next Action Point. For this, the 'Specify User at Previous Action Point' should have been set through Administration: Action Points: Add/Edit. If the Specify User option is not set, there will not be any option to select a User in the Review page. In this case, any one of the users associated with the selected Action Point can Review the work order and take necessary action.

The Previous Review Comments field is listed with all completed action points with corresponding User Name, Action (Outcome), Action Date and the Comments entered by the Reviewer. On reviewing, the user can enter his comments if any, and can add Costs and Invoices if required. At the end of the review, the user can submit the page by choosing the required Action and the next action point user (if the action point is of specific user type). Once the page is submitted, the work order will be moved to the next Action Point as defined in the workflow.

Additional Review Comments can be entered and updated anytime without selecting the Next Action Point User

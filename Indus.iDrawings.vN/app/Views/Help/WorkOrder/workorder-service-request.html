New Requests for the selected Work Type can be created through this option. On clicking this, New Request page will be displayed.

Administrators and Action Point Userscan create and review Requests. Work Order Users can only create New Requests and track its status.

New Request of the selected Work type can be registered by selecting the Work Type, Site and filling up the required fields like Description, Expected Date of Completion, Priority, etc.

There is an option to specify the type of Requester as 'Me' or 'Others'. If the requester is the logged in user, the 'Me' option can be selected. If the request is created for some other person, the 'Other' option can be selected. By default the 'Me' option will be selected. In such cases the First Name, Middle Name, Last Name and email id given when the user is created will be displayed in corresponding fields as read only. If the 'Other' option is selected these fields should be entered manually.

Additional Data fields which are set under Editable Fields (through through Settings: Workflow Settings: Set Work Type: Workflow Settings icon) for the Requester Action Point of the selected Work Type, will be displayed to view/edit.

Site, Building, Floor, Organizational Units drop downs will be populated based on the selections made.

Documents can be attached to the Request by clicking the Manage Documents button. Manage Document will display only if the required permissions are set for the Requester Action point (through Settings: Workflow Settings: Set Work Type: Workflow Settings ) for the selected Work Type. The Request can be saved by clicking the Save button. The saved Request will be available for Review for the action point user under Current Tasks option.

Status of Requests created by logged in user can be tracked through this option. Requests in different status which are completed, closed or Work Order generated also can be viewed through the My Requests option. The two tabs namely Active Requests and Closed Requests are provided to keep track of both currently Active and Closed requests

Edit: Requests which are sent back by the Action Point User can be edited by clicking Edit icon.

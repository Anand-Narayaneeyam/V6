Define Work Types and Set Workflow menu are grouped under Workflow Settings.

Define Work Types

Administrators can create different Work Types for Move Project Work Orders for the execution of Space Planning Move Projects.

On clicking Add icon, New Work Type window will be displayed with Work Type Name, Work Type Code, and Workflow Category. Workflow Category combo will be listed with Workflow categories -‘Space Planning Project Approval’, ‘Move Project Execution’

Add: New Work Type can be added by clicking the Add icon.

Edit: Edit a Work Type by selecting it and then clicking the Edit icon.

Delete: Delete a Work Type by selecting it and then clicking the Delete icon.

For each Work Type, the Action Points in its workflow and the Outcomes (actions) associated with each Action Point have to be defined. Set Workflow option is used to define this.

On selecting a Work Type, any existing Workflow for the selected Work Type will be displayed. The Administrator can modify the Workflow for a Work Type at any time by adding, editing, or deleting new action points as required.

Action Point

Add: This option is used to add Action Points for a selected Work Type along with the Outcomes associated with the Action Point. If the selected Work Type is of the workflow category -Move Project Execution, the Action Point combo will be populated with the Action Points created and mapped for ‘Move Project Execution’ in Workflow Category option in Action Points menu under Administration.

If the selected Work Type is of the workflow category, Move Project Execution, the Action Point combo will be populated with the Action Points created and mapped for ‘Move Project Execution’ in Workflow Category option in Action Points menu under Administration.

Add: New Action point can be added by clicking Add icon.

Edit: Edit an Action point by selecting it and then clicking the Edit icon.

Delete: Delete an Action Point by selecting it and then clicking the Delete icon. Deletion will be possible only if there is no outcome attached to the selected Action Point.

If there is any Work Order in the current Workflow, workflow will be modified when deleting an Action Point. A modification will be considered as revision. When a revision takes place, all existing work orders in that workflow will follow the workflow before the revision.
